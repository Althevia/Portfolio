What is the MVC paradigm? How is it used?

The MVC paradigm is a pattern that connects the user to how the program runs behind the scenes. The three main components of it are model, view, and controller. The controller takes inputs from the user and converts it to commands for the model. The model then manages these commands and alters the view, which the user sees.

What is the difference between a framework and a library?

The framework contains numerous libraries and calls upon them to create basic templates for the user to use.

Why might you use a framework instead of creating an app yourself?

You might use a framework instead because a framework is premade while creating an app would take more time. Being premade, using the framework is much more efficient.
